## Skills — the 1% rule

Skills ship inside plugins. **Invoke the relevant skill before any response or action** — even a 1% chance it applies means use it. See `using-superpowers` for the full rule. These thoughts are red flags meaning "STOP, check for a skill first":

- "This is just a simple question" → check anyway
- "Let me explore the codebase first" → skills tell you *how* to explore
- "The skill feels like overkill" → use it
- "I know what that means" → knowing ≠ using

## Session lifecycle & compaction (the trilogy)

Claude Code fires these hook events during a session. VantaBrain depends on all three being honored so state survives context compaction and session end:

| Event | When it fires | **What you MUST do** |
|-------|---------------|----------------------|
| `SessionStart` | First turn of a session | Already handled: `brain-hook-session` injects kernel + recall + plugins into `additionalContext` so you start briefed. Still run `get_kernel` + `recall vantacode-taxonomies` explicitly for your first response. |
| `PreCompact` | *Before* the conversation window gets summarized (manual `/compact` or auto) | **Call `session_log` with `sync: false`**. Capture `learned`, `errors`, `next` *now* — after compact they're lost. Also call `git_sync` to push brain updates so nothing in-flight gets forgotten. |
| `PostCompact` | *After* compaction completes, receives the summary | **Call `get_kernel` again**. The summary often drops kernel rules, identity, and operating patterns. Re-load them or you'll drift. Also call `recall vantacode-taxonomies` to re-hydrate project context. |
| `Stop` / session end | User ends the session | **Call `session_log` with `sync: true`** — full fields: `project`, `summary`, `learned[]`, `used[]`, `errors[]`, `next[]`. Auto-commits and pushes. One session = one git commit. |

If interrupted (crash, context limit, abort): always call `session_log` with `sync: true` before closing, even with partial `next` items. A committed partial log is better than no log.

**Shortcut**: `/vantabrain compact` runs the PreCompact trilogy manually (session_log → git_sync → prompt you to run `/compact`). Use it any time you're about to compact context.
